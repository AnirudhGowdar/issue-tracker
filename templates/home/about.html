About Issue Tracker

Issue Tracker is an application to track issues or errors in software. Issue Tracker uses a ticketing system to record bugs or other issues in the software development process on a per-project basis. It implements user and role-level security to ensure only authorized users can access tickets and projects.

The application was built using Django framework and Python with an SQLite database. Development was completed with Visual Studio Code .

Bug Tracker has four roles that users can be assigned to.

  • End User
    • Create new tickets for assigned projects
    • View and edit tickets they have submitted
  • Developer
    • Developer can do all functions of End User
    • View and edit tickets they have been assigned
    • Provide solutions to the issues raised by the End User
  • Project Manager
    • Project Manager can do all functions of Developer
    • Create or delete projects
    • Assign users to projects
    • Assign a developer to a ticket
    • Archive projects
  • Admin
    • Admin can do all functions of Project Manager
    • Assign users to roles
    • Remove Users

Some properties of tickets are given below

  • Ticket Priorites
    • Crtical
    • High
    • Medium
    • Low
  • Ticket types
    • Error report
    • Service request
    • Feature request
  • Ticket statuses
    • New
    • Waiting for support
    • Waiting for customer
    • On hold
    • Resolved
